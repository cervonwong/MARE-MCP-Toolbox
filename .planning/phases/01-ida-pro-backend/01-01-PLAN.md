Modify the Dockerfile to add IDA Pro support. Follow the existing Binary Ninja pattern precisely.

1. **Add INSTALL_IDA_PRO build arg** next to the existing INSTALL_BINARY_NINJA arg:
   ```
   ARG INSTALL_IDA_PRO=0
   ```

2. **Add IDADIR env var** to the ENV block:
   ```
   IDADIR=/opt/ida-pro
   ```

3. **Add IDA Pro builder stage** BEFORE the main FROM block. This is a separate FROM stage that extracts and installs IDA Pro from the user-provided archive, so license/installer artifacts never leak into the final image:
   ```dockerfile
   FROM kalilinux/kali-rolling:latest AS ida-builder
   ARG INSTALL_IDA_PRO=0
   RUN --mount=type=bind,from=ida-stage,target=/tmp/ida-stage \
       set -eux; \
       if [ "${INSTALL_IDA_PRO}" != "1" ]; then mkdir -p /opt/ida-pro; exit 0; fi; \
       zipfile="$(find /tmp/ida-stage -maxdepth 1 -name '*.zip' -print -quit)"; \
       if [ -z "${zipfile}" ]; then echo "INSTALL_IDA_PRO=1 but no zip found in ida-stage context" >&2; exit 1; fi; \
       tmpdir="$(mktemp -d)"; \
       unzip -q "${zipfile}" -d "${tmpdir}"; \
       installer="$(find "${tmpdir}" -maxdepth 1 -name '*.run' -print -quit)"; \
       if [ -n "${installer}" ]; then \
         chmod +x "${installer}"; \
         "${installer}" --mode unattended --prefix /opt/ida-pro; \
       else \
         mv "${tmpdir}"/* /opt/ida-pro/ 2>/dev/null || mv "${tmpdir}"/ida* /opt/ida-pro/; \
       fi; \
       rm -rf "${tmpdir}"
   ```
   Key: Support both .run installer and pre-installed directory in the zip. Create empty /opt/ida-pro when INSTALL_IDA_PRO=0 so the COPY --from step doesn't fail.

4. **Add COPY --from=ida-builder** in the final image, after the Binary Ninja install block but before the `detect-it-easy` alias step:
   ```dockerfile
   COPY --from=ida-builder /opt/ida-pro /opt/ida-pro
   ```

5. **Add IDA Pro Python setup block** (conditional on INSTALL_IDA_PRO=1), after the COPY --from step. This installs the idapro Python package from IDA's bundled files, runs the idalib activation script, and installs ida-mcp from PyPI:
   ```dockerfile
   RUN set -eux; \
       if [ "${INSTALL_IDA_PRO}" != "1" ]; then exit 0; fi; \
       python3 -c "import sys; assert sys.version_info >= (3, 12), f'IDA Pro requires Python 3.12+, got {sys.version}'"; \
       ida_python_dir="$(find /opt/ida-pro -type d -name 'python' -path '*/idalib/*' -print -quit)"; \
       if [ -n "${ida_python_dir}" ]; then \
         pip install --no-cache-dir --break-system-packages "${ida_python_dir}"; \
       fi; \
       activate_script="$(find /opt/ida-pro -name 'py-activate-idalib.py' -print -quit)"; \
       if [ -n "${activate_script}" ]; then \
         python3 "${activate_script}" -d /opt/ida-pro; \
       else \
         echo "[warn] py-activate-idalib.py not found -- idalib may not work" >&2; \
       fi; \
       pip install --no-cache-dir --break-system-packages ida-mcp
   ```

6. **Update Ghidra conditional install** -- change the existing condition from checking only INSTALL_BINARY_NINJA to checking both:
   ```dockerfile
   RUN if [ "${INSTALL_BINARY_NINJA}" != "1" ] && [ "${INSTALL_IDA_PRO}" != "1" ]; then \
   ```
   This ensures Ghidra only installs when no commercial disassembler is enabled (per user decision).

7. **Add Python version check** early in the build (after the main package install), as a safety net for INF-03:
   ```dockerfile
   RUN python3 -c "import sys; v=sys.version_info; print(f'Python {v.major}.{v.minor}.{v.micro}')"
   ```

IMPORTANT: Do NOT move the existing Binary Ninja install block. Insert IDA blocks near it but do not alter BN logic.
IMPORTANT: The ida-builder stage must be BEFORE the main `FROM kalilinux/kali-rolling:latest` block (Docker requires builder stages before the stage that references them via COPY --from, or use the separate stage approach).
IMPORTANT: Keep the existing `apt-get install -y --no-install-recommends unzip` capability (already present via the base utils install).

**run_docker.sh changes:**

1. **Add IDA Pro zip detection** right after the Binary Ninja zip detection block (after line ~27):
   ```bash
   # IDA Pro archive: optional for build-time headless install.
   # Place idapro.zip in the repository root, or set IDA_PRO_ZIP explicitly.
   IDA_PRO_ZIP="${IDA_PRO_ZIP:-}"
   if [[ -z "$IDA_PRO_ZIP" && -f "$SCRIPT_DIR/idapro.zip" ]]; then
     IDA_PRO_ZIP="$SCRIPT_DIR/idapro.zip"
   fi
   INSTALL_IDA_PRO=0
   if [[ -n "$IDA_PRO_ZIP" && -f "$IDA_PRO_ZIP" ]]; then
     INSTALL_IDA_PRO=1
     echo "[info] using IDA Pro archive: $IDA_PRO_ZIP"
   else
     IDA_PRO_ZIP=""
     echo "[info] no IDA Pro zip found; building without IDA Pro"
   fi
   ```

2. **Add IDA Pro user directory persistence** after the Binary Ninja user dir block (after the BINARY_NINJA_USER_DIR line ~62):
   ```bash
   # Persist IDA Pro settings/license on the host.
   IDA_USER_DIR="${IDA_USER_DIR:-$HOME/.idapro-docker}"
   mkdir -p "$IDA_USER_DIR"
   ```

3. **Add IDA Pro license seeding** after the Binary Ninja license seeding block. Seed BOTH ida.key and ida.hexlic (IDA 9.x may use either format):
   ```bash
   # Seed IDA Pro license from host if available
   if [[ ! -f "$IDA_USER_DIR/ida.key" && -f "$HOME/.idapro/ida.key" ]]; then
     cp "$HOME/.idapro/ida.key" "$IDA_USER_DIR/ida.key"
     echo "[info] copied IDA Pro ida.key into $IDA_USER_DIR"
   fi
   if [[ ! -f "$IDA_USER_DIR/ida.hexlic" && -f "$HOME/.idapro/ida.hexlic" ]]; then
     cp "$HOME/.idapro/ida.hexlic" "$IDA_USER_DIR/ida.hexlic"
     echo "[info] copied IDA Pro ida.hexlic into $IDA_USER_DIR"
   fi
   if [[ "$INSTALL_IDA_PRO" == "1" && ! -f "$IDA_USER_DIR/ida.key" && ! -f "$IDA_USER_DIR/ida.hexlic" ]]; then
     echo "[warn] no IDA Pro license found in $IDA_USER_DIR" >&2
   fi
   ```
