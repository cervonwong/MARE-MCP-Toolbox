1. Add the new build arg after the existing `INSTALL_BINARY_NINJA` arg (line 8 area):

2. Add an `ida-builder` multi-stage build BEFORE the main FROM (insert at line 2, before the main `FROM kalilinux/kali-rolling:latest`). Use the Docker syntax `FROM kalilinux/kali-rolling:latest AS ida-builder` as a separate stage:
   ```dockerfile
   FROM kalilinux/kali-rolling:latest AS ida-builder
   RUN apt-get update && apt-get install -y --no-install-recommends unzip file && rm -rf /var/lib/apt/lists/*
   RUN --mount=type=bind,from=ida-stage,target=/tmp/ida-stage \
       set -eux; \
       zipfile="$(find /tmp/ida-stage -maxdepth 1 -name '*.zip' -print -quit)"; \
       if [ -z "${zipfile}" ]; then \
         echo "ida-builder: no zip found in ida-stage context, creating empty /opt/ida-pro" >&2; \
         mkdir -p /opt/ida-pro; \
         exit 0; \
       fi; \
       tmpdir="$(mktemp -d)"; \
       unzip -q "${zipfile}" -d "${tmpdir}"; \
       installer="$(find "${tmpdir}" -maxdepth 1 -name '*.run' -print -quit)"; \
       if [ -n "${installer}" ]; then \
         chmod +x "${installer}"; \
         "${installer}" --mode unattended --prefix /opt/ida-pro; \
       else \
         mkdir -p /opt/ida-pro; \
         mv "${tmpdir}"/*/* /opt/ida-pro/ 2>/dev/null || mv "${tmpdir}"/* /opt/ida-pro/; \
       fi; \
       rm -rf "${tmpdir}"; \
       ls /opt/ida-pro/
   ```
   NOTE: When INSTALL_IDA_PRO=0, the ida-stage context will be an empty dir (like binja-stage when BN is disabled), so the builder creates an empty /opt/ida-pro and exits cleanly.

3. In the main FROM stage, after the Binary Ninja install block (after line 87), add the IDA Pro conditional install block:
   ```dockerfile
   # Install IDA Pro headless from a local zip provided as a named build context.
   # Uses multi-stage build: ida-builder extracts the archive, we copy only the
   # installation directory (no installer, no license artifacts in final layer).
   RUN set -eux; \
       if [ "${INSTALL_IDA_PRO}" != "1" ]; then exit 0; fi; \
       echo "[ida] verifying Python >= 3.12"; \
       python3 -c "import sys; assert sys.version_info >= (3, 12), f'Python {sys.version} < 3.12'"; \
       echo "[ida] Python version OK"
   COPY --from=ida-builder /opt/ida-pro /opt/ida-pro
   RUN set -eux; \
       if [ "${INSTALL_IDA_PRO}" != "1" ]; then exit 0; fi; \
       test -d /opt/ida-pro; \
       echo "[ida] installing idalib Python package from bundled files"; \
       idalib_python="$(find /opt/ida-pro -type d -name python -path '*/idalib/*' -print -quit)"; \
       if [ -z "${idalib_python}" ]; then \
         echo "ERROR: could not find idalib/python directory in /opt/ida-pro" >&2; \
         exit 1; \
       fi; \
       pip install --no-cache-dir --break-system-packages "${idalib_python}"; \
       echo "[ida] activating idalib"; \
       activate_script="$(find /opt/ida-pro -name 'py-activate-idalib.py' -print -quit)"; \
       if [ -z "${activate_script}" ]; then \
         echo "ERROR: could not find py-activate-idalib.py in /opt/ida-pro" >&2; \
         exit 1; \
       fi; \
       python3 "${activate_script}" -d /opt/ida-pro; \
       echo "[ida] installing ida-pro-mcp (mrexodia)"; \
       pip install --no-cache-dir --break-system-packages \
         "https://github.com/mrexodia/ida-pro-mcp/archive/refs/heads/main.zip"; \
       echo "[ida] verifying idalib-mcp command is available"; \
       command -v idalib-mcp || { echo "ERROR: idalib-mcp not found after install" >&2; exit 1; }
   ENV IDADIR=/opt/ida-pro
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host filesystem -> Docker build | IDA Pro zip and license files cross from host to build context |
| Host filesystem -> Container runtime | License files cross via bind mount |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Dockerfile ida-builder stage | mitigate | Multi-stage build: IDA installer and license artifacts exist only in ida-builder stage; only /opt/ida-pro directory is COPY'd to final image. No license files, no .run installer in final layer. |
| T-01-02 | Information Disclosure | Docker layer caching | mitigate | IDA zip passed via named build context (bind mount), not COPY. Build context is a temp dir cleaned on exit. No license data in Docker build cache. |
| T-01-03 | Information Disclosure | compose.yaml volume mount | accept | License bind mount (~/.idapro-docker/) is local-only, same risk profile as existing BN license mount. User controls host directory permissions. |
| T-01-04 | Tampering | run_docker.sh IDA zip source | accept | User provides their own IDA zip. Same trust model as Binary Ninja zip -- user is responsible for zip integrity. |
| T-01-05 | Denial of Service | IDA Pro build failure | mitigate | All IDA install steps have explicit error checks with clear messages. Python version check runs before attempting idalib install. Missing idalib/python or activation script causes build failure with actionable error. |
</threat_model>
