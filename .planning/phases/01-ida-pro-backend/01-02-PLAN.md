**Changes to make:**

1. **Replace the existing detection block** (lines 55-89, starting at `# Detect which MCP backend is available` through the closing `fi` of the else/exit 0 block) with a three-way detection block. The new priority order is IDA > BN > Ghidra per user decision:

   ```bash
   # Detect which MCP backend is available.
   # Priority order: IDA Pro > Binary Ninja > Ghidra
   # Only one backend is active at a time (per project decision).
   mcp_name=""
   mcp_command=""
   mcp_args=""
   mcp_env=""

   if [ -d "/opt/ida-pro" ] && [ "$(ls -A /opt/ida-pro 2>/dev/null)" ] && command -v ida-mcp >/dev/null 2>&1; then
     mcp_name="ida_mcp"
     mcp_command="ida-mcp"
     mcp_args="[]"
     mcp_env="{\"IDADIR\": \"/opt/ida-pro\"}"
     echo "[mcp] Using IDA Pro (highest priority installed)"
   elif [ -f /opt/binaryninja/scripts/install_api.py ] && [ -f "${BINJA_ROOT}/binary_ninja_headless_mcp.py" ]; then
     mcp_name="binary_ninja_headless_mcp"
     mcp_command="python3"
     mcp_args="[\"${BINJA_ROOT}/binary_ninja_headless_mcp.py\"]"
     mcp_env="{}"
     echo "[mcp] Using Binary Ninja (IDA Pro not installed)"
   elif [ -f "${GHIDRA_ROOT}/ghidra_headless_mcp.py" ] || { [ -f "${GHIDRA_ROOT}/pyproject.toml" ] && [ -d "${GHIDRA_ROOT}/ghidra_headless_mcp" ]; }; then
     mcp_name="ghidra_headless_mcp"
     mcp_command="python3"
     if [ -f "${GHIDRA_ROOT}/ghidra_headless_mcp.py" ]; then
       mcp_args="[\"${GHIDRA_ROOT}/ghidra_headless_mcp.py\"]"
     else
       mcp_args="[\"${GHIDRA_ROOT}/ghidra_headless_mcp.py\"]"
     fi
     ghidra_install_dir="${GHIDRA_INSTALL_DIR:-}"
     if [ -z "${ghidra_install_dir}" ] && [ -d /usr/share/ghidra ]; then
       ghidra_install_dir="/usr/share/ghidra"
     fi
     if [ -n "${ghidra_install_dir}" ]; then
       mcp_env="{\"GHIDRA_INSTALL_DIR\": \"${ghidra_install_dir}\"}"
     else
       mcp_env="{}"
     fi
     echo "[mcp] Using Ghidra (neither IDA Pro nor Binary Ninja installed)"
   else
     echo "[mcp] warning: no MCP backend (IDA Pro, Binary Ninja, or Ghidra) found, skipping MCP configuration" >&2
     printf '%s\n' '{"mcpServers":{}}' > "${CLAUDE_PROJECT_MCP}"
     write_codex_base_config
     exit 0
   fi
   ```

   Key details for IDA detection:
   - Check /opt/ida-pro exists AND is non-empty (`ls -A` check) because the Dockerfile creates an empty dir when IDA is not installed

   - Set IDADIR in the mcp_env so ida-mcp knows where IDA is installed
   - ida-mcp uses its own command name (not python3 + script path like BN/Ghidra)

2. **Preserve all existing Ghidra detection logic** -- specifically the GHIDRA_INSTALL_DIR detection and env var handling. The Ghidra elif block above is copied verbatim from the current script (lines 66-82), only adding the log message at the end.

IMPORTANT per user decision: If the selected backend fails to start (e.g., license issue), fail with a clear warning -- do NOT silently fall back to the next backend. The detection script picks ONE backend based on what is installed. If it picks IDA but IDA's MCP server later fails at runtime, that is an IDA problem, not a reason to try BN.
